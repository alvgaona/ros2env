## Overview

rosenv manages ROS 2 distributions installed via pixi global. It creates symlinks in `/opt/ros/` for compatibility with standard ROS 2 tools and provides environment isolation when switching between distributions.

## Features

<CardGrid>
  <Card title="Auto-detection" icon="magnifier">
    Scans `~/.pixi/envs/` and automatically detects ROS 2 installations
  </Card>
  <Card title="Symlink Management" icon="link">
    Creates and maintains symlinks in `/opt/ros/` for tool compatibility
  </Card>
  <Card title="Environment Isolation" icon="shield">
    Cleanly switches between distributions with automatic variable cleanup
  </Card>
  <Card title="Diagnostic Tools" icon="setting">
    Built-in verification and troubleshooting commands
  </Card>
</CardGrid>

## Quick Start

<Tabs>
  <TabItem label="macOS">

```bash
# Install ROS 2 with pixi
pixi global install --environment ros-humble-desktop \
  -c robostack-staging ros-humble-desktop

# Setup rosenv
sudo mkdir -p /opt/ros
sudo chown $USER /opt/ros
rosenv setup

# Add shell integration
rosenv init zsh >> ~/.zshrc
source ~/.zshrc

# Activate distribution
rosenv activate humble
```

  </TabItem>
  <TabItem label="Linux">

```bash
# Install ROS 2 with pixi
pixi global install --environment ros-humble-desktop \
  -c robostack-staging ros-humble-desktop

# Setup rosenv
sudo mkdir -p /opt/ros
sudo chown $USER /opt/ros
rosenv setup

# Add shell integration
rosenv init bash >> ~/.bashrc
source ~/.bashrc

# Activate distribution
rosenv activate humble
```

  </TabItem>
</Tabs>

## Why rosenv?

ROS 2 tools expect distributions at `/opt/ros/<distro>`. When using pixi global for installations, rosenv:

- Creates necessary symlinks for compatibility
- Manages environment variables for clean switching
- Provides verification and diagnostic tools
- Works with existing ROS 2 workflows and tutorials
